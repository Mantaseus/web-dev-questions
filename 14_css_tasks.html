  4. CSS selectors: multiple separate classes on the same element

    Task: Style the following divs as shown below.

              <div class="red-bg square margin-below"></div>
          <div class="green-bg rounded square margin-below"></div>
          <div class="blue-bg square circle"></div>

    Here are the various classes we are using

    • The square class defines W=100px and H=100px
    • The rounded class defines a border radius of 10px
    • The circle class turns the square into a circle using a border radius of 50%
    • The margin-below class defines a bottom margin of 8px
    • The red-bg class defines a red background color
    • The green-bg class defines a green background color
    • The blue-bg class defines a blue background color

    Note: Notice how the class names are so easy to read. The classes are also reusable now. See how the square and margin-below classes need to be defined just once but can get used in as many places as we want. A lot of CSS frameworks out there like Semantic UI, Bootstrap, Tailwind CSS, etc. make use of multiple classes to allow users to make their HTML styling cleaner without having to write any CSS at all. They all use the exact same technique that you just used right now.
